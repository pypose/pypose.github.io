pypose.module.LTI

class pypose.module.LTI(A, B, C, D, c1=None, c2=None)[source]

A sub-class of ‘System’ to represent the dynamics of discrete-time Linear Time-Invariant (LTI) system.

Parameters

  • A (Tensor) – The coefficient matrix in the state-space equation of LTI system,

  • B (Tensor) – The coefficient matrix in the state-space equation of LTI system,

  • C (Tensor) – The coefficient matrix in the state-space equation of LTI system,

  • D (Tensor) – The coefficient matrix in the state-space equation of LTI system,

  • c1 (Tensor) – The constant input of the system,

  • c2 (Tensor) – The constant output of the system,

  • state (Tensor) – The state of the current timestamp of LTI system,

  • input (Tensor) – The input of the current timestamp of LTI system.

Returns

The state of the next timestamp (state-transition) and the system output (observation).

Return type

Tuple of Tensors

Every linear time-invariant lumped system can be described by a set of equations of the form which is called the state-space equation.

\[\begin{align*} \mathbf{z} = \mathbf{A}\mathbf{x} + \mathbf{B}\mathbf{u} + \mathbf{c}_1 \\ \mathbf{y} = \mathbf{C}\mathbf{x} + \mathbf{D}\mathbf{u} + \mathbf{c}_2 \\ \end{align*} \]

where we use \(\mathbf{x}\) and \(\mathbf{u}\) to represent state and input of the current timestamp of LTI system.

Here, we consider the discrete-time system dynamics.

Note

According to the actual physical meaning, the dimensions of A, B, C, D must be the consistent, whether in batch or not.

\(\mathbf{A}\), \(\mathbf{B}\), \(\mathbf{C}\), \(\mathbf{D}\), \(\mathbf{x}\), \(\mathbf{u}\) could be a single input or in a batch. In the batch case, their dimensions must be consistent so that they can be multiplied for each channel.

Note that here variables are given as row vectors.

forward(state, input)[source]

Example

>>> A = torch.randn((2,3,3))
    B = torch.randn((2,3,2))
    C = torch.randn((2,3,3))
    D = torch.randn((2,3,2))
    c1 = torch.randn((2,1,3))
    c2 = torch.randn((2,1,3))
    state = torch.randn((2,1,3))
    input = torch.randn((2,1,2))
>>> lti = pp.module.LTI(A, B, C, D, c1, c2)
>>> lti(state,input)
tensor([[[-8.5639,  0.0523, -0.2576]],
        [[ 4.1013, -1.5452, -0.0233]]]),
tensor([[[-3.5780, -2.2970, -2.9314]],
        [[-0.4358,  1.7306,  2.7514]]]))

Note

In this general example, all variables are in a batch. User definable as appropriate.
